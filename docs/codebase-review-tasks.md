# Предложенные задачи по итогам быстрого ревью

## 1) Задача на исправление опечатки
**Проблема:** В `css` есть файл `a_style.css` с неочевидным именем, которое выглядит как случайная опечатка/временный нейминг и ухудшает навигацию по стилям.

**Задача:** Переименовать `css/a_style.css` в осмысленное имя (например, `css/accessibility.css` или `css/theme-overrides.css`), обновить все импорты/ссылки и проверить, что стили продолжают применяться.

**Критерий приёмки:**
- файл переименован;
- нет битых ссылок в `index.html`/`manifest`/других точках подключения;
- визуально приложение не деградировало.

## 2) Задача на исправление ошибки
**Проблема:** В `Search.controller` флаг занятости меняется по пути `/isBusy`, а в остальных местах контроллера и логике загрузки используется `/isLoading`. Из-за этого индикатор загрузки/блокировка UI может работать непредсказуемо при удалении.

**Задача:** Унифицировать состояние загрузки в `controller/Search.controller.js` — использовать один путь (предпочтительно `/isLoading`) во всех async-операциях.

**Критерий приёмки:**
- в удалении и загрузке используется единый state-путь;
- Busy/Loading-индикатор корректно включается и выключается;
- регрессий в сценариях `Delete` и `Retry` нет.

## 3) Задача на исправление комментария/документации
**Проблема:** В `docs/backend-target.md` указано, что fake backend моделирует только create/update/read, хотя в адаптере и fake-сервисе уже реализован и используется `deleteCheckList`.

**Задача:** Актуализировать раздел "Why fake backend exists now" и/или "Mapping strategy to OData V2", явно указав поддержку delete-сценария и соответствующий endpoint (`DELETE /CheckListSet('<id>')`).

**Критерий приёмки:**
- документация отражает фактический CRUD-контракт;
- текст согласован с `BackendAdapter` и `FakeBackendService`.

## 4) Задача на улучшение теста
**Проблема:** Для `SmartSearchAdapter.filterData` нет явных тестов на различие режимов `EXACT` и `LOOSE`, а также на fallback-логику вычисления failed-статуса через `successRate*` при отсутствии `hasFailed*`.

**Задача:** Добавить unit-тесты для `service/SmartSearchAdapter.js`:
- `EXACT`: активные фильтры должны выполняться одновременно (AND);
- `LOOSE`: активные фильтры работают как OR;
- `filterFailedChecks/filterFailedBarriers`: корректная интерпретация `ALL/TRUE/FALSE` при наличии и отсутствии `hasFailed*`.

**Критерий приёмки:**
- тесты покрывают указанные сценарии;
- тесты стабильны и воспроизводимы локально;
- при поломке логики режимов тесты падают с понятным сообщением.
